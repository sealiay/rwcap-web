The output to screen and that to file are different. The screen output (stdout) mainly contains current running status. The file output mainly contains the results of capitance.

The format of screen output: *

Command:  <%command line%>

Preparation costed <%time%>

Online Info. Random Walk Begins

<%for each master net%>
 Online Info. Start For Master <%net name%>

 <%for every 100,000 walks%>
  Online Info.  <%walk number%> random walks, master cap: <%self cap%>, progress <%percentage%>
 <%end for%>

 Master <%net name%> : <%self cap%> +/- <%absolute error%> (<%relative error%>)
<%end for%>

Online Info. Random Walk Ends
Random Walk costed <%time%>
RWCap has run <%walk number%> walks (<%hop number%> hops/walk)
Total cost: <%time%>

The format of file output: *

Command:  <%command line%>

Preparation costed <%time%>

<%for each master net%>
 Master <%net name%> : <%self cap%> +/- <%absolute error%> (<%relative error%>)

 <%for each other net%>
  <%net name%> : <%coupling cap%> +/- <%relative error%>
 <%end for%>

 <%if --extract-level is set to block%>
  block capacitance of <%net name%>'s blocks:

  <%for each block of other net%>
   <%master block%> ↔ <%other net%>::<%other block%> : <%coupling cap%> +/- <%relative error%>
  <%end for%>

 <%end if%>

<%end for%>

Random Walk costed <%time%>
RWCap has run <%walk number%> walks (<%hop number%> hops/walk)
Total cost: <%time%>

* The grammer of above descriptions is like that used in html template. Words inside <%%> are variables or statements (if/for). Others are displayed in the output directly.
